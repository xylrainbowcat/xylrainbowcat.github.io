# 关键算法

计算器最基本的功能就是计算，但对于我们正常输入的表达式，由于要考虑算术优先级等问题，电脑很难对其进行正确的计算，因此我们引入后缀表达式的概念。

# 后缀表达式

如果将我们平时习惯的 将运算符放在两个操作数中间 的表达式称为中缀表达式的话，那么很容易想到后缀表达式的写法就是 将运算符放在操作数后面。

例如： (3 + 4) × 5 - 6 对应的后缀表达式就是 3 4 + 5 × 6 - 。

# 中缀表达式转后缀表达式

那么怎么样才能把我们熟悉的中缀表达式转换为后缀表达式呢？

本项目使用了栈与队列进行求解，具体步骤如下：

从左向右依次读取算术表达式的元素 X，分以下情况进行不同的处理：

  1. 如果 X 是操作数，直接入队
  2. 如果 X 是运算符，再分以下情况：
    1. 如果栈为空，直接入栈。
    2. 如果 X == "("，直接入栈。
    3. 如果 X == ")"，则将栈里的元素逐个出栈，并入队到后缀表达式队列中，直到第一个配对的”(” 出栈。（注："(" 和 ")" 都不 入队）
    4. 如果是其他操作符（+ - * /）, 则和栈顶元素进行比较优先级。 如果栈顶元素的优先级大于等于 X，则出栈并把栈中弹出的元素入队，直到栈顶元素的优先级小于 X 或者栈为空。弹出完这些元素后，才将遇到的操作符压入到栈中。
  3. 最后将栈中剩余的操作符全部入队。

# 后缀表达式的求解

了解了中缀表达式转后缀表达式的方法，还需要知道后缀表达式的求解方法：

首先准备一个栈用于存放结果，接着按照如下步骤进行处理：

  1. 从左开始向右遍历后缀表达式的元素。
  2. 如果取到的元素是操作数，直接入栈，如果是运算符，从栈中弹出 2 个数进行运算，然后把运算结果入栈。
  3. 当遍历完后缀表达式时，计算结果就保存在栈里了。

# 项目设计

# 总体框架

参照微软计算器，本项目将输出分为了两部分，第一部分为 Expression ，用于存放 已输入的表达式，第二部分为 Result ，用于存放 已输入表达式的结果 或者 当前输入的数。
这样输出的好处是可以方便的对当前输入的数进行更改，也进而大大简化了后续拓展功能的实现逻辑。

通过对这种输出模式的深度使用，为了更好的将其复现，本项目给计算器定义了 3 + 1 种状态，分别为 done、toBeInput、normal 以及 error，具体解释如下：

状态解释
done当按下 "=" 时计算器处于 done 状态，此时若按下符号键（如 "+"）或功能键（如 "sin"）会清空表达式并对之前结果进行调用，否则将屏幕清空并执行当前输入的键
toBeInput当按下符号键时计算器处于 toBeInput 状态，此时 "Result" 中显示的为已输入表达式的结果，此时若按下符号键或者功能键则会直接对 "Result" 进行调用，否则将 "Result" 清空并执行当前输入的键
normal当按下数字键（包括小数点）时计算器处于 normal 状态，此时 "Result" 中显示的为当前输入的数，输入其他键会直接对当前结果进行更改
error当出现数学错误时（如除以零）计算器处于 error 状态，此时按下任意一个键都会将屏幕清空，此外 error 状态下的结果不会保存在历史中

注：具体情况比较复杂，上表只列出了每种状态的大致逻辑。

给计算器定义了这四种状态后，后续功能的实现就需要对它们分别进行处理。

# 表达式合法性的检验

在介绍具体按键的实现逻辑之前，我们应先规定表达式合法性检验的逻辑。
对于表达式的合法性检验大致可以分为两种做法：

  1. 不限制用户输入，在计算时判断表达式是否合法。
  2. 限制用户输入正确的表达式。

对于这一点本项目同样参照了微软计算器，采用了后者，限制方式为：若用户输入了错误的表达式，会自动将修正为正确的表达式。
如：用户输入 ((114514 + 3.) 5...3 会自动将其修正为 ((114514 + 3) * 5.3) 。
这种方式可以有效减少计算器报错的概率并简化输入，为用户带来良好的使用体验。

由于在前面介绍过有两部分输出，所以对于合法性的检验也要分别处理，两部分互不干涉，在一定程度上减少了需要处理的情况的数量。
